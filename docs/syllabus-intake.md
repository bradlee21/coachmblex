# Syllabus Intake (Future Paste-In)

Use this doc when class syllabi are provided later.

## What To Paste

Paste only instructor-provided material. Do not infer or invent class order.

Preferred inputs:
- Course title
- Term / cohort (if relevant)
- Ordered module or weekly topic list
- Learning objectives (if included)
- Major exam topics / review guides (if included)

## What We Will Extract

From the provided syllabus, we will map:
- Ordered modules/classes (preserve source order exactly)
- Topic/subtopic lists
- Proposed content-pack boundaries
- Suggested question type mix (`mcq`, `reverse`, `fill`) per pack

## Rules

- Do not invent order.
- Do not invent missing topics.
- If the syllabus is ambiguous, mark items as unknown and wait for clarification.
- Pack planning is data-only until explicitly approved for question drafting/import.

## Where The Plan Lives

- `src/content/syllabus-plan.json`

Update the placeholder entries there once a real syllabus is provided.
